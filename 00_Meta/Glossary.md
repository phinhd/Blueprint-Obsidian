# Glossary

Commonly used terms and definitions in the URL Shortener project.

| Term | Definition |
| --- | --- |
| **Short Code** | The random 6-character string (e.g., `xY7z2Q`) that identifies a short URL. |
| **Alias** | A user-defined, human-readable short code (e.g., `obsidian-guide`). |
| **Long URL** | The original, absolute destination URL provided by the user. |
| **Resolution** | The process of converting a short code back to its original URL and redirecting the user. |
| **Identity Verification** | The process of proving ownership of an email address via a time-limited token. |
| **Refresh Token** | A long-lived token used to obtain a new Access Token without re-authenticating. |
| **Click Event** | A single record representing one successful resolution of a short URL. |
| **Normalization** | The process of cleaning and standardizing URLs or aliases (e.g., lowercase, removing extra slashes). |
